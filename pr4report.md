## Контрольные вопросы с ответами

### 1. Что такое непрерывная интеграция?

Непрерывная интеграция (CI, Continuous Integration) — это практика разработки, при которой изменения в коде регулярно добавляются в общий репозиторий, после чего автоматически выполняются сборка, тестирование и другие проверки проекта.

### 2. Какие задачи решает CI?

CI решает следующие задачи:

1. Автоматическая проверка изменений в коде.
2. Быстрое обнаружение ошибок после коммита.
3. Поддержание стабильного состояния проекта.
4. Снижение количества проблем при объединении веток.
5. Ускорение процесса разработки и поставки изменений.

### 3. Какие существуют системы непрерывной интеграции?

Среди популярных систем непрерывной интеграции можно выделить:

1. Jenkins.
2. GitHub Actions.
3. GitLab CI/CD.
4. TeamCity.
5. Travis CI.
6. CircleCI.

### 4. Что такое Jenkins?

Jenkins — это популярная система автоматизации с открытым исходным кодом, которая используется для организации CI/CD. С её помощью можно автоматически собирать проект, запускать тесты, выполнять проверки качества кода и развёртывание приложения.

### 5. Как установить Jenkins?

Jenkins можно установить разными способами:

1. Локально в операционной системе.
2. Через пакетный менеджер.
3. В Docker-контейнере.
4. В Kubernetes.

В данной работе Jenkins настраивается через Docker, что упрощает установку и запуск.

### 6. Как создать проект в Jenkins?

Для создания проекта в Jenkins необходимо:

1. Открыть веб-интерфейс Jenkins.
2. Нажать `New Item`.
3. Ввести имя проекта.
4. Выбрать тип проекта, например `Pipeline`.
5. Сохранить настройки и перейти к конфигурации.

### 7. Как настроить сборку проекта в Jenkins?

Для настройки сборки проекта нужно:

1. Указать источник кода.
2. Настроить шаги сборки.
3. Добавить команды для установки зависимостей, тестирования и сборки.
4. Сохранить конфигурацию.
5. Запустить проект и проверить результат выполнения.

### 8. Что такое CI/CD-пайплайн?

CI/CD-пайплайн — это последовательность автоматизированных этапов, которые выполняются над проектом: получение исходного кода, сборка, тестирование, анализ и при необходимости развёртывание.

### 9. Как создать CI/CD-пайплайн в Jenkins?

Для создания CI/CD-пайплайна в Jenkins нужно:

1. Создать проект типа `Pipeline`.
2. Описать этапы работы в `Jenkinsfile`.
3. Указать шаги для сборки, тестирования и других операций.
4. Сохранить настройки.
5. Выполнить запуск пайплайна и проверить его прохождение.

### 10. Какие этапы включает в себя CI/CD-пайплайн?

Обычно CI/CD-пайплайн включает следующие этапы:

1. Получение исходного кода.
2. Установка зависимостей.
3. Сборка проекта.
4. Запуск тестов.
5. Проверка качества кода.
6. Публикация артефактов.
7. Развёртывание приложения.

В данной работе реализованы базовые этапы: подготовка окружения, тестирование и сборка проекта.

### Настройка Jenkins через Docker

Для запуска Jenkins используется Docker Compose. Jenkins запускается в контейнере и становится доступен через браузер на порту `8080`.

Пример запуска:

```bash
docker compose up -d --build
```

После запуска Jenkins можно открыть по адресу:

```text
http://localhost:8080
```

### Создание проекта в Jenkins

В Jenkins необходимо создать новый проект типа `Pipeline`. В настройках проекта можно использовать pipeline script из файла `Jenkinsfile`, который находится в репозитории.

### Описание пайплайна

Подготовленный пайплайн включает три основных этапа:

1. `Checkout` — получение исходного кода.
2. `Test` — запуск автоматических тестов командой `go test ./...`.
3. `Build` — сборка проекта командой `go build ./...`.

Таким образом, при запуске pipeline Jenkins автоматически проверяет, что проект собирается и проходит тесты.
